Cifra do Centro — especificação v1.1

Uma linguagem cifrada em português coloquial onde sequências de palavras genéricas codificam valores ASCII via expressões aritméticas.

⚠ birosca (32) nunca codifica espaço — use sempre daí. birosca só aparece em expressões aritméticas.

Regras de avaliação

1.Uma expressão vai de um separador ao próximo. Avalie com precedência: o antes de e / a.
2.Modificadores (dobro, meio…) se aplicam apenas ao próximo token de valor, não à expressão inteira.
3.O resultado da expressão é o código ASCII do caractere (alvo prático: 33–126).
4.vixe antecede o separador do caractere e marca que a letra resultante é maiúscula. Ex: vixe zica e coisa aí … → 'A'.
5.Sem vixe, todo caractere é minúsculo (ou símbolo, se não for letra).
6.Se o caractere resultante não for letra, vixe é silenciosamente ignorado — sem erro. Ex: vixe birosca e coisinha → '!' normalmente.

Nota sobre múltiplas codificações: vários caminhos aritméticos podem produzir o mesmo ASCII — isso é intencional. Não há codificação canônica única; qualquer expressão válida que resulte no valor correto é aceita.

Exemplos de codificação

Caractere: A — ASCII 65

vixe zica e coisa aí = maiúsc( 64 + 2 ) = maiúsc(65) → 'A'

Caractere: a — ASCII 97

dobro birosca e coiso aí = (32×2) + 3 = 64 + 33 = 97 → 'a'

Caractere: ! — ASCII 33

birosca e coisinha aí = 32 + 1 = 33 → '!'

Frase: "Oi!"

vixe dobro birosca e dobro negócio e coiso aí ← 'O' (111, maiúsc) dobro birosca e coisa o bagulho a coiso aí ← 'i' (105) birosca e coisinha né ← '!' (33)
