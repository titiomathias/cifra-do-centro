Faixa de caracteres suportados novo v1.2

A cifra cobre code points 33–252 (ASCII imprimível + Latin-1 estendido). Isso inclui todos os caracteres acentuados e símbolos do português brasileiro.

Para maiúsculas acentuadas (À, Á, Â, Ã, Ç, É, Ê, Í, Ó, Ô, Õ, Ú, Ü), use vixe: a diferença entre minúscula e maiúscula acentuada é sempre 32, igual ao ASCII padrão.

Regras de avaliação

1.Uma expressão vai de um separador ao próximo. Avalie com precedência: o antes de e / a.
2.Modificadores (ué, mó, meio, um monte) se aplicam apenas ao próximo token de valor, não à expressão inteira.
3.O resultado da expressão é o code point Unicode do caractere (faixa alvo: 33–252).
4.vixe antecede o separador do caractere e faz code point − 32. A expressão deve codificar a versão minúscula da letra; vixe converte para maiúscula. Ex: expressão = 97 ('a'), com vixe → 97 − 32 = 65 → 'A'. clarificado v1.2
5.Sem vixe, todo caractere é minúsculo (ou símbolo/acento minúsculo, se não for letra básica).
6.Se o caractere resultante não for letra (após aplicar −32), vixe é silenciosamente ignorado — sem erro.
7.vixe funciona tanto para letras básicas (a–z) quanto para letras acentuadas (à–ü). Em ambos os casos, a diferença maiúsc/minúsc é 32. novo v1.2

Nota sobre múltiplas codificações: vários caminhos aritméticos podem produzir o mesmo code point — isso é intencional. Não há codificação canônica única; qualquer expressão válida que resulte no valor correto é aceita.
